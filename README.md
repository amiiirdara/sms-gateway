# SMS Gateway

A multi-tenant SMS Gateway: send SMS via REST API against a prepaid credit balance, with a guaranteed-SLA "Express" lane and batch "campaign" sending. Built for the ArvanCloud software developer challenge.

- **Design doc:** [ARCHITECTURE.md](ARCHITECTURE.md) - read this first. It covers the reliability model (Outbox/Inbox pattern), the Express SLA policy, campaigns, data model, and REST API contract in full.
- **API contract:** [openapi/openapi.yaml](openapi/openapi.yaml)
- **Agent-facing conventions:** [AGENTS.md](AGENTS.md) and [.cursor/rules](.cursor/rules)

## Status

This is currently a **scaffold**: the full monorepo layout, Postgres schema, ClickHouse schema, OpenAPI contract, Docker Compose stack, and repo conventions (rules/skills) are in place. Every `cmd/` binary builds and runs (with health checks / stub loops), but the actual business logic (balance debit, dispatch, billing, reporting) is not yet implemented - see the `TODO(implementation phase)` comments throughout `cmd/` and the package docs in `internal/`.

## Repo layout

```
cmd/                 One deployable binary per service (see ARCHITECTURE.md section 8)
internal/
  config/            Env-based configuration
  platform/          Infra wrappers: postgres, redis, kafka, clickhouse, inbox, lifecycle
  domain/            Business logic by bounded context: billing, messaging, campaigns, reporting
  db/sqlc/           Generated by sqlc - do not hand-edit
db/
  migrations/        golang-migrate SQL migrations (Postgres schema source of truth)
  queries/           Hand-written .sql compiled by sqlc
clickhouse/init/     ClickHouse table DDL, auto-applied on container start
openapi/openapi.yaml REST API contract
deploy/docker/       Shared Dockerfile for all cmd/ binaries
```

## Prerequisites

- Go 1.22+
- Docker + Docker Compose
- [`golang-migrate`](https://github.com/golang-migrate/migrate) CLI (for `make migrate-up`)
- [`sqlc`](https://sqlc.dev) CLI (for `make sqlc`)
- `make` (Linux/macOS/WSL; on native Windows, run the commands inside the Makefile directly via PowerShell, or use WSL)

## Local development

```bash
# 1. Build every service locally (sanity check)
make build

# 2. Bring up the full stack: Postgres, Redis, Kafka (KRaft), ClickHouse,
#    operator-mock, and every cmd/ service
make docker-up

# 3. Apply the Postgres schema
make migrate-up

# 4. Tail logs
make docker-logs
```

Once running:
- `api-gateway` - http://localhost:8080 (`/healthz`)
- `reporting-api` - http://localhost:8081 (`/healthz`)
- Postgres - localhost:5432 (`sms`/`sms`, db `sms_gateway`)
- Redis - localhost:6379
- Kafka - localhost:9094 (host listener)
- ClickHouse - http://localhost:8123 (HTTP), localhost:9000 (native)

To iterate on a single service against the rest of the dockerized stack, stop that service's container and run it locally instead: `docker compose stop api-gateway && make run-api-gateway` (copy `.env.example` to `.env` first, or export the vars, so it points at `localhost` instead of the in-network hostnames).

Start with [AGENTS.md](AGENTS.md).
